Flags

Flags can be used to set variables which persistent in save files. These are helpful for things like quests, first time events and basically anything custom. Some core features such as triggers use flags.

Setting a flag

Setting a flag by name. Overwrites any existing flag.

game.setFlag('flag-name', anyValueType)

Note: Flags are never destroyed. To unset a flag, simply set the flag to false.

Getting a flag

Getting a flag by name, if the flag does not exist it will return false.
game.getFlag('flag-name')

Temporary Data

Sometimes we require a flag which doesn't need to persist across save files. For example, the text inputs in the game which allow users to search, may retain their search while the game is playing, so that next time it's loaded in - it uses the previous term. Since this is not required across the save file and can be forgotten about when the game is closed, we instead store it in a temporary location which could be cleared at any time and at minimum is cleared when the game is closed.

Note: Unlike flags, temporary data has no expiration set, and will naturally be expired when the application closes or refreshes it state.

Setting temporary data

Setting a temp flag by name. Overwrites any existing temp flag.

game.setTemp('flag-name', anyValueType)

Note: Flags are never destroyed. To unset a flag, simply set the flag to false.

Getting temporary data

Getting a temp flag by name, if the temp flag does not exist it will return false.

game.getTemp('flag-name')
