Blueprints

Blueprints (not to be confused with in-game blueprints or Unreal blueprints) is a data structure in the engine which contains every object definition possible to the game.

All sprites, recipes, functions, UI Components, complex objects are stored in Blueprints before they are initialized.

If you wanted to add an item to the game, for example a sword - you would first add it's blueprint. After that you can give the sword to any player, NPC or container, by passing the blueprint codename to different creation functions.

Syntax

All blueprints are stored on the _BLUEPRINTS global object

Important: You should never directly edit or retrieve a blueprint from the _BLUEPRINTS object, instead use the functions provided below.

_BLUEPRINTS[][]

Get a blueprint

Important: If you modify the blueprint after you get it, there will be side effects

game.getBlueprint(, );

Add a new blueprint

Note: The BLUEPRINT_DEFINITION for each BLUEPRINT_TYPE is different, to check a blueprint definition, simple check an existing definition within that blueprint type.

game.addBlueprint(, );

Blueprints reference

TILESETS

All tileset files

AUDIO

All audio files

SPRITES

All sprite definitions

RECIPES

All crafting recipes

SCENES

All scenes (e.g. front menu, death screen)

COMPONENTS

All UI components (e.g. hud elements, buttons)

PARTICLES

All particle effects

SIMPLE_EXT

All simple extensions (auto-loaded on create)

COMPLEX_ITEMS

All complex items (items that exist in the world)

SPINES

All spine definitions (Advanced Animations)

INV_ITEMS

All inventory items (Including resources)

_STATUSES

All status effects

LOOT_TABLES

All loot tables

ANIMATEDSPRITES

All animated sprites (similar to SPRITES)

DIALOGS

All conversation dialogue

EXTENSIONS

All extensions for COMPLEX_ITEMS

RESEARCH_NODES

All research nodes in research tree
(Auto generated at runtime)
